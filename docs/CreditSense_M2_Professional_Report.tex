% ------------------------- ABSTRACT ------------------------- %
\section*{Abstract}
\addcontentsline{toc}{section}{Abstract}
The traditional loan approval process is hindered by manual document verification, subjective risk evaluation, and bottlenecks in interpreting complex financial jargon. This report presents \textbf{CreditSense v2}, an intelligent, semi-autonomous credit risk assessment platform. 

Built using a hybrid architecture consisting of Streamlit, FastAPI, and a LangGraph-orchestrated Large Language Model (LLM), the platform conducts interactive borrower intake, queries regulatory documents (RBI and NBFC guidelines) via Retrieval-Augmented Generation (RAG), and calculates an explicit risk score utilizing a deterministic policy engine. The result is a bilingual (English and Hindi), highly explainable financial report that empowers loan officers to make rapid and compliant credit decisions.

% ------------------------- 1. INTRODUCTION ------------------- %
\section{Introduction}

In modern financing, decision latency directly impacts customer acquisition and operational overhead. However, completely offloading credit decisions to AI models introduces severe non-compliance risks, as LLMs frequently hallucinate rules or invent financial metrics. 

CreditSense solves this by implementing a **deterministic constraint envelope** over the AI. The LLM is used strictly for conversational data extraction, summarization, and translation, while the actual decision (Approve, Conditional, or Escalate) is grounded in math and fixed policy matrices.

\subsection{Project Scope and Deliverables}
The core deliverables of the CreditSense v2 system include:
\begin{itemize}
    \item A dynamic conversational user interface built on Streamlit.
    \item A LangGraph orchestrator that forces the AI into a strict sequence of operations (Guardrail $\rightarrow$ Conversation $\rightarrow$ Retrieve $\rightarrow$ Score $\rightarrow$ Translate).
    \item A local vector database (ChromaDB) leveraging lightweight ONNX embeddings to retrieve financial regulations.
    \item Formatted, compliant, and bilingual downloadable PDF reports.
\end{itemize}

\subsection{Architectural Advancements: Migrating to ONNX}
A critical requirement for this milestone was ensuring the platform could deploy on lightweight free-tier environments (e.g., Streamlit Community Cloud and Render). 

Initially, the RAG pipeline utilized the standard \texttt{sentence\_transformers} library, which forced a massive (2GB+) PyTorch dependency. This resulted in system crashes during deployment due to memory exhaust. \\
\textbf{The Solution:} We aggressively optimized our storage pipeline by swapping to ChromaDB's native \textbf{ONNX Runtime} (\texttt{ONNXMiniLM\_L6\_V2}). This achieved identical 384-dimensional semantic embeddings while shrinking the dependency footprint from 2.2GB to under 50MB, achieving perfect stability.

% ------------------------- 4. METHODOLOGY -------------------- %
\section{Implementation \& Methodology}

\subsection{LangGraph Node Determinism}
Large Language Models left unchecked are prone to making autonomous, non-compliant decisions. Creating a \texttt{StateGraph} prevents an LLM from leaping directly to a generated report before calculations occur. The orchestrated flow is:
\begin{enumerate}[label=\textbf{Node \arabic*:}]
    \item \textbf{Guardrail:} Validates the conversation block is strictly finance-related.
    \item \textbf{Conversation:} Forces the LLM to extract 15 specific JSON parameters from the user before continuing.
    \item \textbf{RAG Retriever:} Contextualizes the profile by hunting for relevant rules in the RBI corpus.
    \item \textbf{Policy Engine:} Triggers math calculations (Debt-to-Income, Net Loan-to-Value) yielding an integer Score out of 100.
    \item \textbf{Report Generation:} Transcribes the math constraints into natural language.
    \item \textbf{Translate:} Converts the English response into Hindi using LLM prompting.
\end{enumerate}

\subsection{Bilingual PDF Export}
Financial inclusivity requires accessible language. The system implements the \texttt{fpdf2} library paired with custom Unicode-compliant fonts (Noto Devanagari) to output highly formatted, downloadable PDF records of the complete transaction and regulatory citations.

% ------------------------- 5. CONCLUSION --------------------- %
\section{Conclusion \& Future Scope}
CreditSense v2 successfully proves that modern Generative AI can be safely integrated into the highly-regulated fintech sector by combining flexible language chat with rigid, deterministic policy graphs. 

Through the collaborative efforts of the entire project team, the platform transitioned from ideation to a fully functional, cloud-deployed product yielding actionable bilingual credit reports.

\textbf{Future work includes:}
\begin{itemize}
    \item Upgrading the local ChromaDB SQLite implementation to a clustered vector database (e.g., Pinecone or Weaviate).
    \item Implementing formal OAuth2 authentication mechanisms.
    \item Expanding the ML risk classification adapter with real-world historic default datasets.
\end{itemize}
